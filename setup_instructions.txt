# SLG Game Server Framework - Setup Instructions

## Prerequisites

1. **Go 1.19+**: https://golang.org/doc/install
2. **Protocol Buffers Compiler**: https://developers.google.com/protocol-buffers/docs/gotutorial
3. **MongoDB**: https://www.mongodb.com/docs/manual/installation/

## Installation Steps

### Step 1: Install Protocol Buffers Go Plugin
```bash
go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
```

### Step 2: Generate Protocol Buffer Code
```bash
# Navigate to your project directory
cd slg-game

# Generate Go code from .proto files
protoc --go_out=. --go_opt=paths=source_relative proto/messages.proto
```

### Step 3: Install Dependencies
```bash
go mod tidy
```

### Step 4: Start MongoDB
Make sure MongoDB is running on localhost:27017, or update the connection URL in config/game.json

### Step 5: Build and Run
```bash
go build -o slg-server .
./slg-server
```

## Project Structure Overview

The framework is organized as follows:

- **main.go**: Entry point that loads config, initializes DB, and starts the server
- **config/**: Configuration loading and structure definitions
- **proto/**: Protocol buffer message definitions (you'll add more messages here)
- **database/**: MongoDB connection wrapper
- **network/**: TCP connection handling and packet serialization
- **game/**: Core game logic with clear separation of concerns

## Customization Points

### Adding New Messages
1. Edit `proto/messages.proto` to add new C2S/S2C message types
2. Add message ID constants in `game/router.go`
3. Register new handler functions in `NewMessageRouter()`
4. Implement your game logic in the handler functions

### Database Operations
Access MongoDB through the session object in your handlers:
```go
collection := session.db.GetCollection("your_collection_name")
// Use standard mongo-go-driver operations
```

### Configuration
Modify `config/game.json` to adjust server settings like port, database URL, timeouts, etc.

## Example Handler Implementation

In `game/router.go`, you'll find template handlers like this:

```go
func handleLoginRequest(session *PlayerSession, data []byte) *network.Packet {
    // 1. Unmarshal the request
    request := &proto.C2S_LoginRequest{}
    proto.Unmarshal(data, request)
    
    // 2. Implement your business logic here
    //    - Validate credentials
    //    - Query database
    //    - Create player session
    
    // 3. Create and marshal response
    response := &proto.S2C_LoginResponse{
        Success: true,
        Message: "Welcome!",
        PlayerId: 12345,
    }
    responseData, _ := proto.Marshal(response)
    
    // 4. Update session state
    session.SetPlayerID(response.PlayerId)
    session.SetLoggedIn(true)
    
    // 5. Return response packet
    return &network.Packet{
        MsgID: MsgID_S2C_LoginResponse,
        Data: responseData,
    }
}
```

Simply replace the TODO comments with your actual game logic!

## Testing

You can test the server with any TCP client that can send protobuf-encoded messages with the 8-byte header format:
- Bytes 0-3: Message ID (little-endian uint32)
- Bytes 4-7: Message length (little-endian uint32)
- Bytes 8+: Protobuf-encoded message data

The server will automatically route messages to the appropriate handlers based on the message ID.
